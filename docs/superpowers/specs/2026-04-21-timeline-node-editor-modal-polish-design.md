# 时间线节点弹窗样式修正设计

## 背景

上一轮已经将时间线页面中的“创建节点”和“编辑节点”抽成了统一的 `NodeEditorModal` 与 `NodeEditorForm`。但用户在实际界面中反馈了两个具体问题：

- 移动端创建节点弹窗底部的两个操作按钮过大，视觉上接近全宽胶囊按钮，不符合普通表单操作区的尺寸预期。
- web 端的创建节点弹窗虽然复用了新的字段内容，但整体外观仍然保留了 antd `Dialog` 的旧标题/内容节奏，看起来没有完全切换到新的统一样式。

这说明当前统一工作在结构层已经完成，但桌面端外壳与操作区的视觉细节还没有完全收口。

## 目标

- 将移动端节点弹窗底部按钮收敛到正常表单操作区尺寸。
- 保持 web 端使用居中弹窗，但内部头部、正文、按钮区全部改为新的统一壳子。
- 不再让桌面端混用 antd 默认标题栏和默认 footer 风格。
- 保持创建节点与编辑节点继续共用一套组件结构。

## 非目标

- 不改动节点表单字段。
- 不改动创建/编辑的业务逻辑、接口参数、提示文案。
- 不改动时间线卡片、日期快捷编辑、状态快捷编辑等其他交互。
- 不改成自定义桌面蒙层，不重写 `Dialog` 的关闭和定位行为。

## 方案对比

### 方案 1：保留 antd `Dialog` 作为桌面定位壳，内容完全自定义

- 桌面端继续使用 `Dialog` 实现居中弹层与遮罩关闭
- 不使用默认 title/footer
- 所有可见头部、正文、按钮区都由 `NodeEditorModal` 自己渲染

优点：

- 改动小，保留现有可访问性和定位行为
- 桌面端和移动端的内容层级可以完全统一
- 能直接解决“web 端还是老样式”的问题

缺点：

- 需要额外覆盖 antd `Dialog` 的默认内容布局，让旧壳完全退场

### 方案 2：桌面端改为自定义居中浮层

优点：

- 视觉控制完全自主

缺点：

- 会重做焦点管理、遮罩关闭、弹层结构
- 对当前问题来说改动过大，没有必要

## 选定方案

采用方案 1：保留桌面端 `Dialog` 作为外层居中弹层壳，但将可见 UI 全部切换为自定义统一内容。

## 交互设计

### 移动端

- 继续使用底部抽屉式 `Popup`
- 底部操作区仍为左右双按钮
- 按钮尺寸调整为普通表单操作区规格：
  - 高度降低
  - 圆角减小
  - 不再显得像满宽胶囊 CTA

### web 端

- 继续使用居中弹窗
- 保留遮罩关闭与居中显示行为
- 头部、正文、按钮区全部使用 `NodeEditorModal` 内部结构
- 不再显示 antd 默认标题栏和默认 footer 视觉

### 创建 / 编辑模式差异

- 创建节点：
  - 保留 `New Node`
  - 不显示状态字段
  - 主按钮文案 `创建`
- 编辑节点：
  - 保留 `Complete Edit`
  - 显示状态字段
  - 主按钮文案 `保存`

## 组件设计

### `NodeEditorModal`

调整重点：

- 桌面端 `Dialog` 仅作为壳，不承担内容样式表达
- 对话框可见内容统一由：
  - `timeline-node-editor__header`
  - `timeline-node-editor__body`
  - `timeline-node-editor__actions`
 组成

为了让桌面端真正进入新样式，组件需要避免再依赖 antd 的默认 title/footer 渲染方式。

### `NodeEditorForm`

无需改动字段结构，只继续作为表单内容承载组件。

## 样式设计

### 移动端按钮修正

调整 `timeline-node-editor__actions` 下的按钮样式：

- 限制按钮最小高度在正常表单提交按钮区间
- 收紧横向内边距
- 圆角由当前大胶囊感改为更贴近卡片/表单体系的圆角
- 维持左右并列，不改单列

### 桌面端统一新壳

样式目标：

- `Dialog` 外层仍保留圆角、边框、投影
- 默认 header/footer 视觉不再成为主界面的一部分
- 弹窗主体的间距、标题位置、按钮区边界与移动端保持统一语言

### 预期结果

- 移动端创建节点与编辑节点底部按钮视觉尺寸一致且更自然
- web 端创建节点不再像“旧 dialog + 新内容”的混搭
- web 端编辑节点和创建节点外观完全一致，只保留字段差异

## 测试设计

更新 `frontend/src/test/timelineModals.test.tsx`：

- 桌面端创建节点和编辑节点都能找到统一的自定义按钮区
- 桌面端创建节点与编辑节点都继续渲染为 `dialog`
- 移动端创建节点与编辑节点都继续渲染为统一的抽屉容器
- 创建模式不出现状态字段，编辑模式出现状态字段

这次测试重点不是像素级尺寸，而是确保结构与渲染壳已经切换正确。按钮尺寸通过 CSS 修正实现。

## 风险与缓解

### 风险 1：桌面端隐藏默认 `Dialog` 结构后导致内容被裁切

缓解：

- 保留现有 `Dialog` 外层宽高限制
- 只替换内部可见结构，不移除必要的容器滚动能力

### 风险 2：按钮尺寸调小后触控区域不足

缓解：

- 高度只调整到正常操作区级别，不缩到文本按钮大小
- 维持清晰的主次按钮对比

### 风险 3：创建节点与编辑节点再次出现桌面/移动端样式漂移

缓解：

- 继续只通过 `NodeEditorModal` 统一渲染壳
- 不在 `Timeline.tsx` 中重新写回专用容器 JSX

## 验收标准

- 移动端创建节点底部按钮明显比当前版本更小、更正常
- web 端创建节点呈现为新的统一壳样式，不再保留旧 dialog 观感
- web 端编辑节点也使用同样的新壳
- 创建/编辑仍然共用 `NodeEditorModal`
- 现有创建/编辑行为和测试主流程保持通过
