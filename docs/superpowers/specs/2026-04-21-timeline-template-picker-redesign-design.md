# 时间线模板选择弹窗重构设计

## 背景

当前首页空状态中的“选择模板”能力已经可用，但普通用户点击后进入的模板选择弹窗体验较差，主要问题集中在两个方面：

- 底部弹出的进入与退出动画生硬，保留了 `antd-mobile Popup` 的默认抽屉感，不符合当前产品的品牌化移动端体验
- 弹窗首屏直接混合模板列表、模板详情和节点预览，信息层级混乱，内容密度过高，导致用户第一次进入时负担过重

产品目标不是仅仅“修样式”，而是把模板选择能力升级成一个高质量的底部 `sheet` 交互，让空时间线初始化流程在感受上更像一个完整、自然、可控的引导步骤。

## 目标

- 保留底部弹出的产品形态，不改成独立页面或居中对话框
- 将模板选择流程重构为分层体验：首屏选择模板，第二层查看模板详情
- 优化弹出、切换、关闭动画，使其更顺滑、更接近原生移动端 sheet
- 提升视觉层级、可读性和操作稳定性
- 保持现有模板应用逻辑与 API 不变，确保本次工作聚焦在普通用户端的交互体验升级

## 非目标

- 不改模板数据结构
- 不改模板应用接口和后端权限逻辑
- 不改管理员模板管理页
- 不新增拖拽关闭、手势返回、复杂物理动画
- 不在本轮中引入全新的全局弹层系统

## 核心方案

采用“品牌化底部 Sheet + 面板内双层视图”的方案。

用户点击首页空状态中的“选择模板”后，不再直接看到一整个长列表，而是进入一个经过重构的底部模板面板：

- 第一层为模板列表视图，只负责“选择哪套模板”
- 第二层为模板详情视图，展示当前模板的简介与节点预览
- 底部提供固定操作区，确保用户在浏览内容时始终有稳定的返回与确认动作

这样可以显著降低首次进入时的理解成本，也能把动画、层次和品牌表达统一到一个组件中。

## 交互结构

### 1. 打开入口

入口保持不变：

- 仅在时间线为空时出现
- 位于首页空状态卡片中
- 按钮名称仍为“选择模板”

### 2. 弹层结构

弹层整体采用接近全屏高度的底部 sheet，但仍保留顶部留白，使其明显区别于独立页面。

内部结构由四部分组成：

- 顶部头部区
- 视图主体区
- 底部固定操作区
- 遮罩层

### 3. 视图层级

弹层内部包含两个显式视图状态：

- `list`：模板列表视图
- `detail`：模板详情视图

打开弹层时默认进入 `list`。

#### `list` 视图

展示内容：

- 标题与辅助说明
- 模板卡片列表

模板卡片展示：

- 模板名称
- 模板简述
- 节点数量
- 少量节点标签或预览摘要

交互规则：

- 点击模板卡片进入 `detail`
- 选中态需要明确，但不在首屏直接展开完整节点列表

#### `detail` 视图

展示内容：

- 返回按钮
- 当前模板名称与简介
- 节点预览列表

节点预览改成步骤列表，而不是现有的统计块样式。每个节点只强调两层信息：

- 节点名称
- 预置描述

当节点没有预置描述时，不再使用大号“无预置说明”，改为弱化辅助文案，例如：

- `暂未配置预置说明`

### 4. 底部操作区

在 `detail` 视图下始终显示底部固定操作区。

按钮建议为：

- `返回`
- `使用此模板`

其中：

- `返回` 用于回到 `list`
- `使用此模板` 触发现有模板应用逻辑

底部操作区不跟随内容滚动，避免用户浏览较长节点预览后失去行动出口。

## 动画设计

### 1. 打开动画

打开动画拆分为两层：

- 遮罩先快速淡入
- 面板主体再进行轻微上浮与缩放恢复

建议方向：

- 位移距离控制在较短范围内，避免“整页被推上来”的感觉
- 时长控制在约 200ms 到 240ms 之间
- 缓动采用前快后稳的曲线

目标感受：

- 不是“默认抽屉滑上来”
- 而是“一个带重量感的面板浮起并停稳”

### 2. 关闭动画

关闭动画与打开方向相反：

- 面板先轻微下沉并收束
- 遮罩随后淡出

需要避免遮罩先消失导致面板像被硬切掉。

### 3. 视图切换动画

`list -> detail` 与 `detail -> list` 不使用整页硬切。

建议采用轻量内容切换动画：

- 当前内容淡出并微位移
- 新内容淡入并就位

目标是让用户感知到“面板内部进入下一层”，而不是“整个弹窗重新渲染”。

## 视觉设计

### 1. 整体定位

视觉语言应与当前时间线首页保持一致，但比普通表单弹窗更精致，强调：

- 婚礼筹备产品的品牌感
- 柔和但有层次的容器质感
- 明确的主次关系

### 2. 头部

头部需要具备以下元素：

- 顶部拖拽条样式的视觉锚点
- 标题
- 辅助说明
- 关闭按钮

头部的职责是告诉用户：

- 当前动作是“选择一套初始模板”
- 这不是普通确认框，而是一次初始化决策

### 3. 模板卡片

模板卡片需要形成明显主次层级：

- 当前主推荐或选中卡片更突出
- 其他卡片保持克制

重点信息层级：

- 模板名称为主
- 模板说明为辅
- 节点数量与标签作为快速判断信息

### 4. 详情列表

节点预览采用清单式排版：

- 左侧可有编号或步骤标记
- 右侧为名称与描述

避免以下现状问题：

- 重复的大字号文本
- 每个节点都像统计面板一样抢视线
- 长列表没有阅读节奏

### 5. 底部操作条

底部操作区建议使用悬浮或半透明承托感设计，使其明显独立于内容滚动区域。

目标是让用户始终知道：

- 现在看的是内容
- 底部是决定动作

## 组件设计

本次重构重点落在 `frontend/src/pages/Timeline/TimelineTemplatePicker.tsx`。

建议将其重构为明确的视图状态组件，而不是继续在一个平铺结构中同时渲染列表和详情。

### 1. 本地状态

组件至少维护以下状态：

- `view: 'list' | 'detail'`
- `activeTemplateId`
- `selectedTemplate`

打开弹窗时：

- 默认进入 `list`
- 默认高亮首个模板卡片或当前已选模板

点击卡片后：

- 切到 `detail`
- 若详情未加载则调用现有详情接口

### 2. 结构拆分

组件内部建议拆成以下职责块：

- `SheetHeader`
- `TemplateCardList`
- `TemplateDetailPreview`
- `TemplateActionBar`

可以先以内联子组件或局部结构拆分实现，不强制一轮拆成多个文件，但要避免继续堆成单个长 JSX。

### 3. 数据流

数据流保持现有接口模式：

- 打开弹窗时加载模板列表
- 点击模板时加载模板详情
- 点击“使用此模板”时调用现有 apply 接口

应用成功后行为保持不变：

- 关闭弹窗
- 刷新时间线
- 首页退出空状态

## 边界条件

### 1. 加载态

需要针对两个层面分别处理加载：

- 模板列表加载中
- 模板详情加载中

不建议继续使用单行“模板加载中...”文字作为主要反馈，至少需要更明确的骨架或占位结构。

### 2. 空模板列表

如果当前没有可用模板：

- 弹窗仍应保持完整容器结构
- 主文案明确告知“当前暂无可用模板”
- 用户仍可关闭弹窗，继续使用手动创建节点流程

### 3. 应用中状态

点击“使用此模板”后：

- 主按钮进入 loading / disabled 状态
- 避免重复点击造成多次提交

### 4. 错误提示

模板详情加载失败或模板应用失败时：

- 保持面板上下文不丢失
- 用现有 `Toast` 或同类提示透出错误

## 测试要求

### 1. 前端交互测试

需要补充或更新以下覆盖：

- 空时间线点击“选择模板”后默认进入模板列表视图
- 点击模板卡片后进入详情视图
- 详情视图可返回列表视图
- 节点无描述时展示弱化占位文案，而不是旧的大号文案
- 点击“使用此模板”仍能正确调用应用接口并刷新时间线

### 2. 回归测试

需要确保以下行为不退化：

- 时间线为空时仍显示“选择模板”入口
- 时间线非空时仍隐藏“选择模板”入口
- 模板应用成功后模板入口消失

## 实施顺序

建议按以下顺序实现：

1. 重构 `TimelineTemplatePicker` 的视图状态与结构
2. 新增或调整模板选择弹窗专用样式
3. 补上列表视图与详情视图切换动画
4. 更新测试并做移动端回归验证

## 成功标准

- 用户点击“选择模板”后，首屏看到的是清晰的模板选择列表，而不是长内容堆叠
- 模板详情和节点预览具备更清晰的阅读节奏
- 弹层打开、关闭、层内切换都显著比当前顺滑
- “使用此模板”的确认路径更稳定、更易理解
- 模板应用逻辑与空状态入口逻辑保持正确
