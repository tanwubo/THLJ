# 节点预算与工作台行为调整设计

## 背景

当前节点详情页和时间线存在以下问题：

- 节点预算没有独立数据结构，无法在节点维度维护预算并与实际支出做对比。
- 节点详情汇总信息与业务目标不一致，缺少预算展示，且不需要继续展示结余。
- 节点预算需要在节点详情页直接交互编辑，而不是通过独立表单或待办维度录入。
- 费用与收入弹层当前要求分类和描述，超出本次需求，录入应收敛为仅填写金额。
- 附件上传后只能显示文件项，缺少可预览、可点击放大的图片体验，也缺少非图片文件的打开路径。
- 当节点内全部待办完成时，系统没有引导用户同步流转节点状态到已完成。
- 时间线节点列表与节点内待办列表没有按截止时间从近到远排序。
- 节点状态展示存在折行，影响可读性。
- 行内编辑取消行为需要统一规范，统一以 `Esc` 取消。

## 目标

本次调整仅覆盖节点预算、工作台交互、排序与预览行为，不扩展到预算拆分、预算历史或待办级预算。

完成后系统应满足：

- 每个节点只有一个预算金额。
- 节点详情页顶部仅展示预算、总支出、总收入三项。
- 节点详情页预算可点击后行内编辑，并支持 `Enter` 保存、失焦保存、`Esc` 取消。
- 总支出超过预算的 80% 时，高亮总支出展示项。
- 费用和收入录入只填写金额。
- 图片附件支持缩略图预览与点击查看大图；非图片附件点击后触发下载，由用户在本地查看。
- 当节点内全部待办完成时，提示是否将节点状态流转为已完成，用户确认后执行流转。
- 时间线节点和节点内待办均按截止时间从近到远排序。
- 节点状态展示保持单行，不允许折行。

## 非目标

- 不实现待办级预算。
- 不实现多条预算记录或预算历史。
- 不实现费用分类、备注增强或预算使用图表。
- 不实现复杂附件预览器，非图片文件只走下载。

## 方案选型

### 方案 A：在节点表增加 `budget` 字段

推荐采用该方案。

原因：

- 预算天然属于节点，不与费用流水混淆。
- 与“每个节点只有一个预算金额”的需求完全一致。
- 节点详情与时间线读取链路最短，前后端改动集中。
- 后续做预算预警时计算简单，避免把预算塞进费用表造成语义污染。

### 方案 B：在费用表里增加预算类型

不采用。

原因：

- 预算不是实际收入或支出流水。
- 汇总与排序逻辑会混淆预算和真实费用。
- 前端工作台与统计代码会出现大量特判。

## 数据设计

### 数据库

在 `timeline_nodes` 表增加：

- `budget DECIMAL(10,2)`，允许为空，默认视为 `0`

迁移逻辑要求幂等：

- 新库建表时直接包含 `budget`
- 老库通过 schema 补丁增加该字段

### 接口

以下接口需要返回 `budget`：

- `GET /timeline`
- `GET /timeline/:id/workbench`
- `POST /timeline`
- `PUT /timeline/:id`

以下接口需要支持写入 `budget`：

- `POST /timeline`
- `PUT /timeline/:id`

预算值规则：

- 前端以字符串输入，保存前转换为数字。
- 空字符串保存为 `0`。
- 非数字、负数、无限值判定为非法输入并拒绝保存。

## 页面与交互设计

### 1. 时间线列表

节点列表排序规则调整为“截止时间从近到远”：

- 有截止时间的节点按日期升序排列。
- 无截止时间的节点排在最后。
- 截止时间相同则按原 `order` 升序兜底。

节点卡片需要继续展示截止时间，并保证状态 pill 不折行：

- 状态容器使用不换行样式。
- 过长文本时优先保证 pill 单行显示，不压缩到两行。

### 2. 节点详情页汇总区

顶部统计区只展示三项：

- 预算
- 总支出
- 总收入

规则：

- 预算直接取节点 `budget`
- 总支出为该节点下所有待办 `expense` 类型金额之和
- 总收入为该节点下所有待办 `income` 类型金额之和
- 当预算大于 `0` 且总支出大于预算的 80% 时，高亮“总支出”

高亮样式要求：

- 与普通卡片同布局，不改变卡片尺寸
- 通过文字颜色、底色或描边强调风险，不使用闪烁或动画

### 3. 节点预算行内编辑

预算展示区域可交互：

- 默认态展示 `预算` 标签和金额文本
- 点击后切换为输入框
- 输入框聚焦并选中文本
- `Enter` 保存
- 失焦保存
- `Esc` 取消并恢复编辑前值

统一规范：

- 本次涉及的所有行内编辑取消行为都通过 `Esc` 实现

保存结果：

- 成功后更新本地状态并提示成功
- 失败则回退到原值并提示错误

### 4. 节点内待办排序

待办列表排序规则调整为“截止时间从近到远”：

- 有截止时间的待办按日期升序排列
- 无截止时间的待办排在最后
- 截止时间相同按创建时间倒序兜底，保持较新的待办优先

### 5. 费用与收入录入

当前费用录入弹层收敛为最小输入：

- 选择类型：支出 / 收入
- 输入金额

移除：

- 分类
- 描述

保存校验：

- 金额必填
- 金额必须为大于 `0` 的合法数字

后端兼容策略：

- `category` 不再由前端输入
- 后端创建费用时根据类型自动写入默认分类
- 建议默认值：
  - `expense` -> `其他支出`
  - `income` -> `其他收入`

这样可兼容现有表结构，避免本次为移除字段再做数据库变更。

### 6. 附件预览与下载

附件区改成可交互文件项：

- 图片文件显示可用缩略图
- 图片缩略图或文件项点击后打开图片预览层
- 预览层显示大图，支持关闭
- 非图片文件点击后触发浏览器下载

交互规则：

- 删除按钮需要阻止冒泡，避免误触预览或下载
- 图片加载失败时显示图片占位态，仍允许点击原文件地址
- 非图片不做内嵌预览

下载策略：

- 使用附件真实 URL 和 `download` 行为触发保存
- 若浏览器在新标签打开，由用户本地查看，不在应用内做额外接管

### 7. 待办完成后的节点流转

当用户将某个待办标记为完成后，重新计算节点内待办状态：

- 若仍有未完成待办，不触发节点流转提示
- 若全部待办均已完成，则弹出确认框

提示文案方向：

- “当前节点内待办已全部完成，是否将节点状态同步为已完成？”

确认结果：

- 用户确认：调用节点更新接口，将节点状态改为 `completed`
- 用户取消：仅保留待办完成状态，不修改节点状态

边界情况：

- 若节点本身已经是 `completed`，不重复提示
- 若节点内没有待办，不触发该提示
- 只有从“存在未完成待办”切换到“全部完成”时提示，避免刷新后反复弹出

## 实现拆分

### 后端

- 为 `timeline_nodes` schema 增加 `budget`
- 扩展节点创建、更新、查询逻辑以读写 `budget`
- 工作台查询保留原有费用与附件聚合，但待办 SQL 改为支持新的排序策略
- 时间线查询结果改为按截止时间优先排序
- 费用创建逻辑允许前端只传金额和类型，自动补默认分类

### 前端

- 扩展 `TimelineNode`、`NodeWorkbench` 等类型定义以包含 `budget`
- 时间线页面按新规则渲染排序后的节点
- 节点详情页汇总区改成预算、总支出、总收入
- 新增预算行内编辑状态与保存逻辑
- 待办列表按新规则排序
- 费用弹层简化为仅金额输入
- 附件区增加图片预览与非图片下载交互
- 待办完成后增加“流转节点状态”确认逻辑
- 状态 pill 样式修正为单行

## 错误处理

- 预算保存失败时恢复编辑前值，不保留脏状态
- 金额输入非法时前端立即提示，不发请求
- 附件预览失败时保留文件项和下载能力
- 节点自动流转失败时不回滚已完成的待办状态，仅提示节点状态更新失败

## 测试策略

### 后端测试

- schema 补丁会新增 `timeline_nodes.budget`
- 节点创建与更新可正确读写 `budget`
- 时间线结果按截止时间排序，无截止时间排末尾
- 工作台待办按截止时间排序，无截止时间排末尾
- 费用创建在缺少分类时自动补默认分类

### 前端测试

- 节点详情页展示预算、总支出、总收入，不再展示结余
- 总支出超过预算 80% 时出现高亮样式
- 预算行内编辑支持点击编辑、`Enter` 保存、失焦保存、`Esc` 取消
- 附件图片点击后能打开预览层
- 非图片附件点击触发下载行为
- 全部待办完成时出现节点完成确认框，确认后调用节点状态更新
- 时间线节点与待办列表按截止时间排序
- 状态 pill 保持单行展示

## 风险与取舍

- 费用表仍保留 `category`、`description` 字段，但前端不再暴露，属于兼容性保留；这是有意的最小改动。
- 时间线当前依赖 `order` 维护顺序，本次展示层改为截止时间优先后，用户看到的视觉顺序可能与原手工顺序不同；这是需求要求，保留 `order` 仅作兜底。
- 图片预览依赖上传后的公开 URL 可访问；若部署环境静态资源映射异常，预览仍会失败，需要依赖现有 `/public/uploads` 暴露链路正常工作。
